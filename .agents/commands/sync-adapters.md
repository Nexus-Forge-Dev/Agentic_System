# Command: /sync-adapters
# .agents/commands/sync-adapters.md
# Owner: Orchestrator
# Trigger: /sync-adapters
# Run after: any change to MANIFEST.md, PROJECT.md, rules/global.md

---

## Purpose
Regenerate all tool-compatibility shim files from the canonical .agents/ source.
CLAUDE.md, AGENTS.md, and .cursor/rules/*.mdc are outputs — never edited manually.

---

## Workflow

```
STEP 1 — Read canonical sources
  Read .agents/MANIFEST.md     → routing table, division registry, MCP registry
  Read .agents/rules/global.md → 12 Ironclad Rules (condensed form)
  Read .agents/PROJECT.md      → stack, conventions, hard rules
  Read .agents/commands/*.md   → list of all skills (names + descriptions only)
  Read .agents/mcp/settings.json → tool registry (for adapter tool reference sections)
  Enumerate .agents/personas/*.md → full persona roster count + division mapping
    (Confirms: orchestrator, engineering-lead, frontend-developer, backend-architect,
     database-engineer, platform-lead, devops-engineer, cloud-architect, security-engineer,
     incident-commander, quality-lead, sdet, performance-tester, visual-qa-specialist,
     qa-automation-engineer, design-lead, ui-designer, ux-researcher,
     design-systems-engineer, animator, intelligence-lead, session-analyst,
     optimization-architect, research-moderator, research-advocate, research-skeptic,
     research-devils-advocate, research-domain-expert = 28 personas)

STEP 2 — Generate CLAUDE.md (for Claude Code)
  Write to: <project-root>/CLAUDE.md
  Structure:
    # <Project Name> — Agent System
    [DO NOT EDIT — auto-generated by /sync-adapters from .agents/]
    Last synced: <ISO timestamp>

    ## How This System Works
    <2-paragraph summary: division hierarchy + skill routing>

    <routing table from MANIFEST.md — condensed>

    <12 rules in brief form>

    ## Project Stack & Conventions
    <stack + conventions from PROJECT.md>

    ## Key Commands
    <all commands with one-line descriptions>

    ## Pre-Flight
    <8-step pre-flight sequence — brief form>

STEP 3 — Generate AGENTS.md (for OpenCode + OpenAI Codex)
  Write to: <project-root>/AGENTS.md
  Structure:
    # <Project Name> — Agent Instructions
    [DO NOT EDIT — auto-generated by /sync-adapters]
    Last synced: <ISO timestamp>

    ## Codebase Overview
    <from PROJECT.md — what this project does + stack>

    ## How to Run / Test
    <run commands from PROJECT.md>

    <routing table + command list>

    <naming, file structure, conventions>

STEP 4 — Generate .cursor/rules/agents.mdc (for Cursor)
  Write to: <project-root>/.cursor/rules/agents.mdc
  Structure:

  Write to: <project-root>/.cursor/rules/backend.mdc

STEP 7 — Confirm sync
  Report to user: "Adapters synced. Updated: CLAUDE.md, AGENTS.md,
  .cursor/rules/agents.mdc, .cursor/rules/frontend.mdc, .cursor/rules/backend.mdc"

STEP 8 — Update MANIFEST.md System State
  Update the `Last /sync-adapters` field in MANIFEST.md System State section
  with the current ISO timestamp.
  This makes MANIFEST the authoritative record of when adapters were last regenerated.
```

---

## Hard Rule
CLAUDE.md, AGENTS.md, and .cursor/rules/*.mdc must begin with:
  `[DO NOT EDIT — auto-generated by /sync-adapters from .agents/ — Last synced: <ISO>]`
This prevents accidental manual edits to generated files.
