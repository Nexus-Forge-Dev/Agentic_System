# Command: /office-hours
# .agents/commands/office-hours.md
# Owner: UX Researcher (Design Division)
# Trigger: /office-hours "<idea or goal>"

## Purpose
Product reframing session. Challenge assumptions. Define the actual problem
before any design or code work begins. Surface hidden requirements and scope risk.

## Protocol

```
STEP 1 — Listen
  Restate the user's goal in your own words to confirm understanding.

STEP 2 — Reframe
  State the underlying problem (may differ from stated goal).
  "What I heard: <paraphrase>. The deeper problem seems to be: <reframe>"

STEP 3 — Ask the 3 hard questions (one at a time, wait for answers)
  Q1: "Who SPECIFICALLY is this for? Not 'everyone' or 'users' — name a person."
  Q2: "What pain are they experiencing TODAY without this?"
  Q3: "How will we know in 30 days if this worked? What changes?"

STEP 4 — Map the user journey
  Sketch the core flow in text (not visuals):
  User: opens app → does X → sees Y → achieves Z

STEP 5 — Define MVP boundary
  "These are the MUST-HAVEs for v1:"
  "These are NICE-TO-HAVEs we're deferring:"

STEP 6 — Surface risks
  What could go wrong? What are we assuming that might be false?

STEP 7 — Write brief
  Write: docs/office-hours-brief.md
  Contents: problem statement, target user, success metrics, MVP scope, risks, open questions
```

## Output Artifacts
- `docs/office-hours-brief.md`
